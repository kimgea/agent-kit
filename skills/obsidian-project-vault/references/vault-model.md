# Vault Model

Use three layers:

- source
  - `05 Intake` for active staging
  - `65 Sources` for processed raw source material kept for fallback reference
- thinking
  - `10 Ideas`
  - `20 Explorations`
- durable
  - `30 Designs`
  - `40 Requirements`
  - `50 Decisions`
  - `60 Reference`
  - `70 Project Areas`

Key rules:

- do not treat intake or coordination notes as canonical truth
- do not force everything into one giant design document
- keep durable conclusions in designs, requirements, decisions, or reference notes
- use area notes in `70 Project Areas` to connect related work across note types

Core coordination notes:

- `00 Start Here/Start Here.md`
- `01 Coordination/Current Focus.md`
- `01 Coordination/Handoff Log.md`
- `50 Decisions/Decision Index.md`
