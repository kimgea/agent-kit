# Vault Operating Model

## Purpose

This vault is the shared memory layer for `{{project_name}}`.

It is meant to preserve:

- promising ideas
- dead ends worth remembering
- multiple competing designs for the same problem
- requirements derived from design work
- decisions and rationale
- useful reference material

## Working Model

Use three layers:

1. `source`
   - active intake and retained source material
2. `thinking`
   - ideas, explorations, comparisons, alternatives
3. `durable`
   - designs, requirements, decisions, reference

## Folder Intent

### 05 Intake

Use for material that exists but is not yet normalized into the vault.

### 65 Sources

Use for raw source material that has already been processed into the vault but should still remain available as historical or supporting reference.

### 10 Ideas

Use for discrete ideas that may be promising but are not yet serious designs.

### 20 Explorations

Use for alternatives, comparisons, experiments, and failed attempts worth retaining.

### 30 Designs

Use for structured design documents with a defined problem, approach, tradeoffs, and open questions.

### 40 Requirements

Use for implementation-facing constraints.

### 50 Decisions

Use for decision records with rationale and alternatives.

### 60 Reference

Use for stable knowledge that supports multiple areas.

### 70 Project Areas

Use one folder or index note per major topic and link outward to all related work.
